# PointRead

PointRead 是一个面向 Windows 的轻量点读助手。

它的目标很简单：当屏幕上的文字看不清、不能复制、或者读起来费眼睛时，可以用快捷键框选一块区域，让程序识别并朗读出来。

## 功能特性

- **框读**：按快捷键后拖拽框选屏幕区域，使用 OCR 识别并朗读。
- **选读**：在支持标准文本选区的应用中，尝试直接读取当前选中文字。
- **剪贴板朗读**：先复制文字，再让 PointRead 朗读剪贴板内容。
- **播放控制**：支持暂停、停止、后退 5 秒、前进 5 秒。
- **语速调节**：以 `0.9x`、`1.0x`、`1.1x` 这类倍率显示。
- **快捷键设置**：框读、选读、剪贴板朗读和播放控制快捷键都可以配置。
- **托盘常驻**：关闭主窗口后仍可在后台运行。
- **悬浮控制窗**：提供框读、剪贴板朗读和播放控制入口。
- **高 DPI 适配**：自动根据系统缩放换算截图坐标，并提供手动校准。

## 默认快捷键

| 功能 | 快捷键 |
| --- | --- |
| 框读 | `Alt + Q` |
| 选读 | `Alt + E` |
| 剪贴板朗读 | `Alt + W` |
| 暂停 / 继续 | `Alt + P` |
| 后退 5 秒 | `Alt + Left` |
| 前进 5 秒 | `Alt + Right` |
| 停止 | `Alt + S` |

## 适用场景

- 网页正文、文档、PDF、聊天窗口中的文字阅读
- 图片、扫描件、截图中的文字 OCR 朗读
- 复制一段文字后快速朗读
- 对高分屏或缩放环境下的文字进行辅助阅读

## 当前限制

- OCR 识别效果取决于系统 OCR、字体清晰度、背景复杂度和截图区域。
- “选读”依赖目标应用是否暴露标准文本选区。记事本和网页通常可用，微信、部分自绘应用可能不可用。
- 当前语音使用系统基础朗读能力，自然度后续还可以继续提升。
- 后退 / 前进 5 秒是基于文本长度估算的位置跳转，不是音频播放器级别的精确时间轴。

## 开发环境

- Windows 10/11
- .NET SDK 9
- WPF

构建：

```powershell
dotnet build .\PointRead\PointRead.csproj
```

在构建后，可以在.\PointRead\bin\Debug\net9.0-windows10.0.19041.0目录下打开PointRead.exe进行体验


运行：

```powershell
dotnet run --project .\PointRead\PointRead.csproj
```

如果 `.NET` 首次运行权限受限，可以临时指定 CLI 工作目录：

```powershell
$env:DOTNET_CLI_HOME = (Join-Path (Get-Location) '.dotnet-home')
dotnet run --project .\PointRead\PointRead.csproj
```

## 开源初衷

PointRead 起源于一个很具体的需求：希望在 Windows 上拥有一个比浏览器朗读插件更通用、更顺手的点读工具。

它不是完整读屏器，而是一个轻量辅助阅读工具。我们希望它能把“看得见但读起来费劲”的文字，变成可以随手听到的内容。
